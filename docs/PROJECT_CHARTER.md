# Project Charter — PGMS

## Name

**PGMS** — **Postgraduate Management System**

## Purpose

PGMS will become a clean postgraduate resident and supervisor management system.

The project starts as a new clean-room software project, not as a continuation of the previous PGSIMS implementation.

## Why reset

The previous implementation accumulated useful work, but the visible product became confusing due to legacy routes, multiple workflow surfaces, hidden backend features, and paused bridge modules.

The new project will start with a clean scaffold and then redefine every model and workflow one by one.

## Non-negotiable principle

No old model, route, API, workflow, database data, or frontend structure is accepted automatically.

## Initial pilot target

Future pilot target may include:

- 2 departments
- approximately 10 supervisors
- approximately 30 residents

But even these pilot workflows must be freshly designed and accepted before implementation.

## Brick 0 goal

Create only the technical scaffold and documentation system.

## Brick 0 acceptance boundary

- No domain apps
- No custom user model
- No copied legacy code
- No business workflows
- No role-specific UI
- No production data model
